- **Location:** `server/channels/app/webhook.go::doOutgoingWebhookRequest()`
- **Endpoint:** `POST /api/v4/hooks/outgoing`
- **Impact:** Cloud credential theft, internal network access, potential RCE


### Key Evidence

The vulnerability exists because Mattermost makes HTTP requests to user-controlled callback URLs **without any validation**:

```go
// No URL validation before making request
resp, err := a.Srv().outgoingWebhookClient.Do(req)
```

**Attackers can:**
1. Create webhook pointing to `http://169.254.169.254/latest/meta-data/` (AWS credentials)
2. Trigger webhook via message
3. Receive cloud metadata in response
4. Compromise entire cloud infrastructure
